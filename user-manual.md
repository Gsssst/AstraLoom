## 1. 快速开始

### 1.1 部署系统

系统由管理员部署在服务器上。如果你是管理员，请参考 `README.md` 中的 Docker Compose 部署步骤。普通用户只需浏览器访问即可。

### 1.2 注册账号

1. 打开浏览器，访问系统地址（由管理员提供，如 `http://your-server`）
2. 在首页点击右上角的 **「登录」** 或直接点击 **「注册」**
3. 填写用户名、邮箱和密码，点击 **「注册」**
4. 注册成功后自动跳转到应用主界面

![注册流程]

### 1.3 登录

1. 访问系统首页，点击 **「登录」**
2. 输入用户名/邮箱和密码
3. 登录成功后进入应用主界面

> 💡 **提示：** 系统会在 Token 过期时自动刷新，无需频繁登录。

## 3. AI 对话

对话模块是系统最常用的功能入口。你可以在这里与大模型自由对话，也可以启用知识库检索和网络搜索来增强回答。

### 3.1 创建新会话

1. 点击侧边栏 **「对话」** 进入聊天页面
2. 首次使用时系统会自动创建一个空会话
3. 点击左侧会话列表顶部的 **「＋」** 按钮创建新会话

### 3.2 发送消息

1. 在底部输入框输入你的问题
2. 按 `Enter` 发送（`Shift+Enter` 换行）
3. 等待 AI 流式回复

### 3.3 使用提示词快捷方式

输入框上方有 5 个快捷提示词按钮：

| 按钮 | 功能 |
|------|------|
| 总结论文 | 快速生成论文总结提示词 |
| 找研究 Gap | 分析研究空白 |
| 对比方法 | 对比不同论文的方法 |
| 生成 Idea | 提出创新性研究想法 |
| 润色文本 | 将文本润色为学术风格 |

### 3.4 启用知识库检索 (RAG)

- 开启 **RAG 开关**（默认开启）
- AI 回答时会自动从你的论文库中检索相关内容作为参考
- 回答中会标注引用来源

### 3.5 启用网络搜索

- 开启 **网络搜索开关**
- 选择搜索深度：
  - **快速**：1 个搜索查询，适合简单事实查询
  - **标准**（推荐）：3 个搜索查询，平衡速度与覆盖面
  - **深度**：5 个搜索查询，适合需要全面了解的话题
- 搜索结果会以 `[WEB-N]` 格式标注来源

### 3.6 检索策略组合

| RAG | 网络搜索 | 效果 |
|-----|---------|------|
| ✅ 开 | ❌ 关 | 仅从本地知识库检索 |
| ❌ 关 | ✅ 开 | 仅联网搜索 |
| ✅ 开 | ✅ 开 | 混合检索（推荐） |
| ❌ 关 | ❌ 关 | 仅依靠模型自身知识 |

### 3.7 文件上传

支持上传以下类型的文件：

- **图片**（PNG、JPG 等）：Base64 编码后发送给多模态模型分析
- **PDF**：自动提取文本内容
- **文本文件**（TXT、MD 等）：直接读取内容

点击输入框左侧的 **「📎」** 按钮上传文件。

### 3.8 管理会话

- **切换会话**：点击左侧会话列表中的条目
- **重命名会话**：鼠标悬停在会话标题上，点击编辑图标
- **删除会话**：点击会话旁的删除按钮
- **搜索会话**：在会话列表顶部的搜索框输入关键词

### 3.9 思考过程展示

如果开启了思考模式（在设置中配置），AI 的推理过程会显示在可折叠的 **思考面板** 中，包含实时计时器。

### 3.10 空状态引导

当没有任何会话时，系统会展示 4 个引导卡片，帮助你快速开始：
- 提问讨论
- 上传论文
- 检索知识库
- 脑暴灵感

## 4. 行动中心

行动中心是系统的智能调度中枢。它会自动扫描各模块状态，为你生成具体可执行的下一步行动建议。

### 4.1 查看行动建议

1. 点击侧边栏 **「⚡ 行动中心」**
2. 页面按四大模块分组展示行动项：
   - **论文与知识库**：补全文、补向量、继续阅读等
   - **研究方向**：创建方向、评审草稿 Idea 等
   - **写作助手**：继续草稿、创建新草稿等
   - **项目空间**：查看看板、创建空间等

### 4.2 行动优先级

每个行动项标注优先级：

| 标签 | 含义 | 示例 |
|------|------|------|
| 🔴 **优先处理** | 影响核心流程，建议立即处理 | 缺少全文影响问答效果 |
| 🟡 **建议推进** | 值得推进，但不紧急 | 有未读论文等待阅读 |
| 🔵 **可安排** | 可择机处理 | 查看空间看板 |

### 4.3 执行行动

- **导航类行动**：点击 **「进入」** 跳转到对应页面
- **API 类行动**：点击 **「执行」** 直接触发后端操作（如补全文、补向量），执行完成后自动刷新状态

### 4.4 典型使用场景

> 📋 **场景：新安装系统初始化**
> 1. 进入行动中心 → 看到「先建立论文基底」高优先级建议
> 2. 点击「进入」→ 跳转论文库开始搜索和导入论文
> 3. 完成后回到行动中心 → 可能看到「补向量以提升检索命中」
> 4. 点击「执行」→ 系统自动为缺失向量的论文生成嵌入

## 5. 项目空间

项目空间是 v2 版本最核心的新增功能，将论文、研究方向、写作草稿和团队成员整合到统一的协作工作台。

### 5.1 创建项目空间

1. 点击侧边栏 **「📦 项目空间」**
2. 点击 **「新建空间」**
3. 输入空间名称（例如："Video Grounding 综述与实验"）
4. 填写描述（可选，建议补充研究目标、当前阶段和协作范围）
5. 点击 **「创建」**

### 5.2 空间列表页

每个空间卡片展示：

- **角色标签**：owner（紫色）/ editor（蓝色）/ viewer
- **成员数**：当前协作人数
- **推进进度条**：自动计算的科研推进进度
- **资源计数**：已绑定的论文数、方向数、写作项目数

### 5.3 空间详情页

点击空间卡片进入详情页，包含以下区域：

#### 启动台（顶部引导卡片）

三个步骤引导你搭建空间：
1. **绑定核心论文**（推荐下一步）—— 选择空间相关的关键论文
2. **创建或绑定方向**（可执行）—— 将分类论文沉淀成研究方向
3. **推进写作项目**（可选）—— 开始草稿撰写

#### 看板统计卡片

展示论文数、研究方向数、写作项目数、最近活动等关键指标，每个卡片带有状态标签（就绪/需补充）。

#### 资源面板

分三列展示已绑定的论文、研究方向、写作草稿，支持点击跳转到详情页。

### 5.4 绑定资源到空间

在空间详情页的 **「绑定空间资源」** 区域：

1. 选择资源类型（论文 / 研究方向 / 写作草稿）
2. 搜索你的资源（按标题/描述）
3. 点击候选资源旁的 **「绑定」** 按钮
4. 或使用 **「手动输入 ID」** 模式直接粘贴资源 ID

### 5.5 管理团队成员

1. 在空间详情页右侧 **「成员」** 卡片中点击 **「添加」**
2. 输入对方的用户名或邮箱
3. 选择角色：
   - **viewer**：只读成员，可查看空间内容
   - **editor**：编辑者，可绑定/解绑资源
4. Owner 可以随时移除成员

### 5.6 从资源页面关联空间

在论文详情页、研究方向页或写作项目页：
- 右侧会显示 **「项目空间」** 面板
- 展示当前资源已加入的空间
- 可以一键加入或移出可用空间

### 5.7 活动日志

空间详情页右侧展示最近操作的时间线：
- 谁在什么时间创建/更新了空间
- 谁添加/移除了成员
- 谁绑定/移除了资源

## 6. 论文库

论文库是系统的知识管理中心。在这里你可以搜索、发现、导入和管理学术论文。

### 6.1 搜索论文

论文库支持两种搜索模式：

#### 远程搜索（发现新论文）

1. 在搜索框输入关键词（支持中英文）
2. 选择搜索来源：
   - **综合学术检索**（推荐）：同时查询 arXiv + Semantic Scholar + OpenAlex + Google Scholar
   - **arXiv**：仅搜索预印本
   - **Semantic Scholar**：侧重高引论文和引用信息
   - **OpenAlex**：侧重期刊/会议论文
3. 可选筛选：年份范围、排序方式
4. 点击搜索，查看结果列表

> 💡 **搜索技巧：**
> - 英文关键词效果更好（如 "video grounding" 而非 "视频定位"）
> - 结果少时放宽年份范围
> - 每个来源提供的结果可能不同，建议先用「综合学术检索」
> - 结果列表会显示每篇论文的来源标签和来源链接，方便追溯

#### 本地搜索（查找已入库论文）

- 将搜索来源切换为 **「本地」** 即可在已入库论文中搜索
- 支持关键词搜索和语义搜索

### 6.2 导入论文到本地库

在搜索结果中，每篇论文旁边有 **「导入」** 按钮：

1. 点击 **「导入」**，论文元数据（标题、作者、摘要等）立即入库
2. 可勾选 **「自动下载 PDF」**，系统会从 arXiv 获取全文
3. 导入成功后可继续在论文详情页下载 PDF 或查看提取的文本

### 6.3 论文收藏与阅读管理

在论文列表中，每篇论文支持以下操作：

- ⭐ **收藏/取消收藏**：将论文加入个人收藏
- **共享标记**：将论文标记为 **重点论文** 或 **有趣论文**，标记会对所有用户可见，适合团队共同识别高价值论文
- **阅读状态标记**：
  - 📖 **待读**（unread）
  - 🔄 **阅读中**（reading）
  - ✅ **已完成**（completed）
- **论文文件夹**：将论文分配到自定义文件夹中管理

> 个人收藏只影响自己的列表；共享标记写在论文本身上，团队中其他人打开论文库或论文详情页也能看到。

### 6.4 论文文件夹

1. 在论文库页面可以创建文件夹来组织论文
2. 文件夹支持嵌套结构
3. 将论文拖入对应文件夹进行分类管理
4. 文件夹可直接作为研究方向种子，用于生成科研 Idea

### 6.5 批量导入

- **BibTeX 导入**：上传 `.bib` 文件，系统自动解析并创建论文记录
- **Zotero CSV 导入**：上传从 Zotero 导出的 CSV 文件

### 6.6 分类系统

论文支持多级分类树，系统会自动为导入的论文推荐分类标签。你也可以手动管理分类。

### 6.7 封面推荐

当你收藏论文形成一定规模后，系统会分析收藏覆盖度：
- 识别已覆盖的主题和缺失的主题
- 推荐补充论文以填补研究缺口
- 提供 classic（经典）、recent（最新）、gap（缺口）、related（相关）四类推荐

## 7. 论文推送中心

推送中心让你无需手动搜索即可每天收到基于兴趣的 arXiv 论文推送。

### 7.1 设置推送订阅

1. 进入 **「设置」** → **「论文推送」** 标签页
2. 输入关注关键词（如 `video grounding, temporal action localization, multimodal`）
3. 开启 **「站内推送」**
4. 选择推送时间（默认早上 8:00）
5. 点击 **「保存并测试」** 可立即生成一条测试推送验证效果

### 7.2 查看推送

1. 点击侧边栏 **「📚 论文库」** → 点击页面上方 **「论文推送中心」** 入口
2. 或在论文库中点击推送通知
3. 每条推送包含：
   - LLM 生成的当日摘要
   - 推荐论文列表（含标题、作者、年份、推荐分数、推荐理由）
   - 每篇论文的摘要片段

### 7.3 处理推送论文

每篇推送论文提供 7 种操作：

| 操作 | 说明 |
|------|------|
| 📎 **查看来源** | 打开原始论文页面 |
| 📄 **打开 PDF** | 直接查看 PDF |
| 📥 **加入论文库** | 导入到本地知识库 |
| 📖 **加入待读** | 加入待读列表 |
| ▶️ **开始阅读** | 入库并跳转论文详情页 |
| 👍 **感兴趣** | 标记为感兴趣（优化推荐） |
| 🕐 **稍后阅读** | 入库并标记为待读 |
| 🚫 **不感兴趣** | 减少此类推荐 |

> 💡 **推荐反馈循环**：你标记的「感兴趣」和「不感兴趣」会持续优化未来推送的排序，让推荐越来越精准。

## 8. 研究方向

研究方向模块是系统的核心创新引擎。它基于你的论文库中的证据，通过多阶段流水线自动生成有据可依的研究思路。

### 8.1 创建研究方向

1. 点击侧边栏 **「🔬 研究方向」**
2. 点击 **「新建方向」**
3. 填写：
   - **方向名称**（如 "Temporal Action Localization"）
   - **描述**：简要说明研究目标
   - **关键词**：用逗号分隔，用于检索相关论文
4. 可选：选择论文种子（从论文库中选取参考论文）或论文收藏夹
5. 点击 **「创建」**

### 8.2 生成研究 Idea

在研究项目详情页：

1. 点击 **「生成 Idea」** 启动流水线
2. 系统会自动执行 7 个阶段：
   - 📋 **简报**：编制项目背景简报
   - 🔍 **检索**：从本地论文库和外部源收集证据
   - 🕳️ **缺口映射**：LLM 分析研究空白
   - 💡 **生成**：三条路径并行生成候选假设
   - 🔄 **去重**：基于语义相似度合并重复思路
   - ⭐ **评审**：六维度打分（新颖性、证据支撑、可行性、可测试性、影响力、清晰度）
   - ✅ **选择**：输出 Top-N 最优方案
3. 每个阶段进度通过 SSE 实时展示

> 💡 **Idea 三路径生成：**
> - **grounded**：从研究缺口直接推导 → 最可靠
> - **inspiration**：跨论文组合灵感 → 最具创新性
> - **seed_refinement**：基于你提供的种子思路细化 → 最符合预期

### 8.3 查看和评审 Idea

生成的每个 Idea 包含：
- **假设陈述**：核心研究假设
- **方法概述**：建议的研究方法
- **新颖性分析**：与现有工作的区别
- **证据图谱**：支撑该思路的论文证据
- **六维评审得分**：各维度打分和评审意见
- **实验计划**：数据集、基线、评估指标、实验步骤

### 8.4 Idea 演进

1. 点击 Idea 卡片上的 **「演进」** 按钮
2. 填写反馈（如实验初步结果、新发现的局限）
3. 系统基于反馈生成改进的子版本
4. 每个 Idea 维护完整的演进谱系树（祖先/后代关系）

### 8.5 Idea 验证循环

1. 对高评分 Idea 进行多轮验证
2. 每轮验证基于新证据评估 Idea 的可行性
3. 支持从草案到验证通过的状态追踪

### 8.6 生成实验代码

1. 在 Idea 详情页点击 **「生成代码」**
2. 系统生成 PyTorch 代码骨架
3. 包含模型定义、数据处理、训练循环等

### 8.7 比较多个 Idea

1. 选中多个 Idea
2. 点击 **「比较」**
3. 系统生成并排对比表：假设、方法、优劣、适用场景

### 8.8 分享研究方向

1. 在研究项目页点击 **「分享」**
2. 生成分享链接
3. 团队成员可通过链接查看研究方向内容

## 9. 写作助手

写作助手提供从引文推荐到完整草稿生成的全流程辅助，包含 V1 快速工具和 V2 多智能体流水线两种模式。

### 9.1 写作页面布局

进入写作页面后，有两个主标签页：

- **📝 写作项目**：项目管理、章节级 LaTeX 编辑、编译预览、AI 辅助写作
- **🔧 写作工具**：独立写作工具（快速使用）

### 9.2 快速写作工具（V1）

在「写作工具」标签页中有 8 个工具：

| 工具 | 功能 | 使用方式 |
|------|------|----------|
| **引文推荐** | 基于文本推荐相关论文引用 | 输入段落文本，选择论文范围 |
| **相关工作生成** | 自动生成 Related Work 章节 | 输入主题，可选生成对比表格 |
| **文本润色** | 学术风格润色 | 粘贴文本，选择风格（学术/简洁/流畅/英文） |
| **摘要生成** | 结构化摘要生成 | 输入标题和关键点 |
| **文献综述** | 完整文献综述 | 输入主题，生成综述+对比表 |
| **论文对比** | 多篇论文方法/结果对比 | 输入论文 ID 列表 |
| **小组报告** | 生成 .docx 小组会议报告 | 选择论文列表 |
| **基金申请** | NSFC 申请书辅助 | 撰写/评审/提炼创新点/润色 |

### 9.3 写作项目管理（V2）

#### 创建写作项目

1. 切换到 **「写作项目」** 标签页
2. 点击 **「新建项目」**
3. 选择或绑定论文项目上下文；如果需要会议格式，可在预览模式中选择单栏、双栏或模板样式
4. 可选：绑定到研究方向（从研究上下文继承论文引用）

#### 章节级 LaTeX 写作

1. 在写作项目中创建章节，例如 Introduction、Method、Experiments
2. 每个章节都可以直接编辑 LaTeX 源码
3. 预览区会合并所有章节，显示接近完整论文的 PDF 预览
4. 输入 `\` 后会出现常用 LaTeX 命令提示，如 `\cite{}`、`\ref{}`、`\section{}`
5. 引用建议会优先来自论文库和项目证据卡

#### references.bib 与 figures/

- **references.bib 面板**：根据已有证据卡自动生成 BibTeX，可查看、复制，也可用于 LaTeX 引用
- **figures/ 面板**：维护图表清单，生成 `figure` 插入片段，方便复制到章节中
- 文件结构中的 `references.bib` 和 `figures/` 可以点击定位到对应面板

#### 多智能体写作流水线

启动流水线后，5 个专业智能体按顺序执行：

1. **Selector（选择器）**：从知识库检索相关论文
2. **Reader（阅读器）**：阅读选定的论文，提取关键信息
3. **Writer（写作器）**：根据 7 种任务特定提示词生成内容
4. **Reviewer（评审器）**：从清晰度/完整性/正确性/引用四维度评审
5. **Citation（引文器）**：格式化引用、交叉验证、生成 BibTeX

- 进度通过 PipelineProgress 组件实时展示
- 不同任务类型调用不同的智能体组合

### 9.4 差异润色（Diff Polish）

1. 在章节编辑器中选中文本，点击 **「润色」**
2. 系统生成润色后的版本，并展示行级差异对比
3. 你可以：
   - 🟢 **接受** 单个修改块
   - 🔴 **拒绝** 单个修改块
   - 查看版本历史

### 9.5 引文决策循环

1. AI 生成引文后，系统自动搜索知识库验证
2. 真实存在的引文标记 🟢 绿色
3. 无法验证的引文标记 🔴 红色，并提供替换建议
4. 你的确认和修正反馈会持续提升引文准确性

### 9.6 草稿质量教练

1. 在写作项目中点击 **「质量评估」**
2. 系统从多维度评估草稿：
   - 结构完整性
   - 论证逻辑
   - 引用覆盖率
   - 语言质量
3. 给出具体改进建议和优先修复项

### 9.7 导入/导出

- **LaTeX 导入**：上传 `.tex` 文件，自动提取章节结构和参考文献
- **LaTeX 编译预览**：支持单栏、双栏和模板样式预览；服务端需要安装 `pdflatex`
- **多格式导出**：Markdown、LaTeX、Word .docx

## 13. 最佳实践建议

### 13.1 新手入门路径

推荐按以下顺序逐步探索系统：

```
1. 设置推送订阅 → 2. 搜索导入论文 → 3. 论文库收藏分类
    ↓
4. 创建项目空间 → 5. 绑定论文资源 → 6. 创建研究方向
    ↓
7. 生成研究 Idea → 8. 使用写作助手 → 9. 查看行动中心
```

### 13.2 高效的论文管理流程

1. **每日早上**：查看推送中心的每日论文推送，标记感兴趣/不感兴趣
2. **精读论文**：在论文详情页使用 AI 问答深入理解论文
3. **团队共识**：把真正关键的论文标记为「重点论文」，把启发性强但未必核心的论文标记为「有趣论文」
4. **分类归档**：用文件夹和分类系统组织论文
5. **定期维护**：在行动中心检查全文和向量覆盖情况

### 13.3 研究方向最佳实践

1. 先积累至少 10-20 篇相关论文再启动 Idea 生成
2. 选择高质量的种子论文作为起点
3. 对生成的 Idea 进行多轮演进，而非一次性接受
4. 利用比较功能筛选最有潜力的方向

### 13.4 协作建议

1. 为每个课题创建独立的项目空间
2. 将核心论文绑定到空间，团队成员共享阅读
3. 邀请合作者作为 editor，共同管理空间资源
4. 定期查看空间活动日志，了解团队进展

### 13.5 写作建议

1. 先用 V1 快速工具做片段润色和引用推荐
2. 确定方向后用 V2 多智能体流水线生成完整草稿
3. 使用差异润色功能逐块审阅修改
4. 开启引文决策循环确保引用准确性
5. 在章节级 LaTeX 编辑器里合并预览全文，检查单栏/双栏/模板效果
6. 最后用草稿质量教练做整体评估

## 附录：常见问题

### Q: 搜索结果为空怎么办？
- 尝试更宽泛的英文关键词
- 切换到「综合学术检索」来源
- 检查年份范围是否过窄

### Q: 论文全文加载失败？
- 部分论文可能无开放获取 PDF（paywall）
- 尝试切换 arXiv PDF 镜像源（管理员配置）
- 论文仍可基于摘要进行问答

### Q: Idea 生成效果不理想？
- 确保论文库中有足够的种子论文（建议 10+ 篇）
- 检查论文全文覆盖率（缺少全文会影响证据提取）
- 尝试调整研究项目的关键词和描述

### Q: 如何让推送更精准？
- 使用更具体的关键词（如 "video temporal grounding" 而非 "video"）
- 积极标记推送论文的反馈（感兴趣/不感兴趣）
- 定期更新关键词以匹配研究兴趣变化
