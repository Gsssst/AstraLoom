## Context

The README should work as a long-lived project homepage. Detailed one-time upload instructions are useful during repository setup, but they make the public README look less polished after the repository is already published.

## Approach

- Remove the `GitHub Safety Checklist` section.
- Remove the Chinese `上传到 GitHub 时要注意` section.
- Keep short warnings near Quick Start and Deployment Notes about not committing `.env`, API keys, uploaded files, private papers, or database backups.

## Non-Goals

- Do not change `.gitignore`.
- Do not change setup commands.
- Do not add a new docs page in this change.
