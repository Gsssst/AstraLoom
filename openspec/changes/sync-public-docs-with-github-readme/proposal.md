## Why

The README now presents AstraLoom as a GitHub-ready, lab self-hosted research workspace, but the deeper public documentation still mixes older positioning, stale frontend template text, and less precise guidance. This creates confusion for new lab maintainers who read beyond the README.

## What Changes

- Update `introduction.md` to match the current AstraLoom positioning, architecture, module map, and data boundaries.
- Update `user-manual.md` so lab admins and members have a current, practical guide for paper library, toolbox, research direction, writing, settings, and safe operation workflows.
- Replace the default Vite text in `frontend/README.md` with AstraLoom frontend development notes.
- Refresh `openspec/project.md` so OpenSpec project context matches the current product name, model support, and lab self-hosting intent.
- Keep `README.md` as a concise GitHub project homepage and do not reintroduce a long upload checklist.
