## Context

Papers are stored globally in `papers`, while per-user saved/read state is stored in `user_papers`. The requested "我的" view is about who imported the paper into the library, not who saved it later, so ownership should be stored on the paper record itself. Existing rows must display as imported by `gst`.

Group meeting reports are generated by `ReportService.generate_report()` and exported to Word in `app/api/writing.py`. The current prompt is hard-coded per paper and the Word export sets only a generic Normal font.

## Goals / Non-Goals

**Goals:**
- Persist importer account metadata for paper records.
- Apply importer metadata consistently across remote ingestion, personal ingestion, BibTeX import, and Zotero CSV import.
- Add a current-user "我的" filter to paper search and display importer tags on paper cards.
- Preserve existing global/local/source search modes.
- Allow users to replace or extend group-report prompt instructions from the UI.
- Set Word export fonts to SimSun for Chinese and Times New Roman for Latin text.

**Non-Goals:**
- Change paper visibility or authorization semantics; this is a filter/tag, not row-level privacy.
- Rework the entire paper ownership model into per-user paper copies.
- Build a full report-template management system.

## Decisions

- Add nullable `imported_by_user_id` and `imported_by_username` columns to `papers`.
  - `imported_by_user_id` enables accurate filtering for known users.
  - `imported_by_username` preserves a display label and supports the historical `gst` backfill even if no matching user row exists.
- Backfill existing papers with `imported_by_username='gst'`.
- Thread an optional `imported_by_user` object into ingestion service calls rather than reading auth state inside the service.
- Add `owner=mine` to local paper search; remote search ignores owner because remote previews are not local imports.
- Treat custom report instructions as a bounded text field appended to the default context. The service still supplies title, abstract, and full text so users do not need to rewrite all prompt boilerplate.
- Centralize Word font application in helper functions in `writing.py` so headings and body paragraphs receive both Latin and eastAsia font settings.

## Risks / Trade-offs

- Historical `gst` backfill may not map to a real user id -> use username label and only filter "我的" by id when the current user matches the imported id; username fallback covers the `gst` account if that user exists.
- Duplicate paper reuse means a later user adding an already-imported paper will not become the original importer -> this preserves the "入库账号" meaning as first importer.
- Custom instructions can produce unexpected report structures -> still bound length and keep paper context available.
