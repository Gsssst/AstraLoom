## Why

写作工作台已经能展示项目状态和下一步动作，但写作项目创建时仍偏“空白项目”。用户已经在论文库分类和研究方向里沉淀了大量上下文，如果写作项目不能直接绑定这些上下文，后续生成 Related Work、Introduction 和引用检查仍需要重复手动搬运资料。

## What Changes

- 写作项目创建支持绑定研究方向、论文分类、目标会议/年份和写作类型。
- 后端将绑定分类中的论文合并为写作项目证据来源，并记录上下文元数据。
- 论文写作项目创建弹窗提供上下文选择，创建后项目工作台能够显示这些上下文。
- 上下文项目会预填更明确的 Introduction、Related Work、对比表和 References 起点。

## Impact

- 写作项目 API 请求模型扩展，但保留旧字段兼容。
- 写作项目服务新增上下文解析与项目预填逻辑。
- 前端写作项目创建面板需要加载研究方向和论文分类。
