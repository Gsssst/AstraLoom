## Context

Mature GitHub READMEs such as Dify, Open WebUI, Langflow, and Paperless-ngx usually open with a concise positioning statement, badges, a feature overview, quick-start commands, and self-hosting guidance. Diagrams help readers understand how the product fits together before reading long setup details.

## Approach

- Put a concise hero section at the top.
- Use bilingual Chinese/English content, but avoid duplicating every paragraph in full length.
- Add a table of contents for scanning.
- Use Mermaid diagrams because GitHub renders them natively and no image assets are needed.
- Keep safety and upload guidance compact and explicit.

## Non-Goals

- Do not create generated image assets.
- Do not add screenshots until a stable release UI screenshot is chosen.
- Do not change runtime configuration or deployment behavior.
